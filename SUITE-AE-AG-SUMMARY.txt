Graceful Degradation:
  • If a report URL is not found → Mark as GAP (feature not in scope)
  • If test data is missing → Mark as SKIP (document reason)
  • If report crashes → Mark as FAIL (escalate to development)
  • Empty date ranges → Must show graceful message (not crash)

Test Outcomes:
  PASS:  All steps completed; report loaded, generated, displayed correctly
  FAIL:  Test step failed with error; report did not load; app crashed
  SKIP:  Test data not available OR precondition failed
  GAP:   Report feature does not exist in app (404 or menu item not found)

Success Metrics:
  ✓ At least 12 TCs PASS (out of 15 total)
  ✓ No more than 2 TCs FAIL (critical bugs)
  ✓ GAP results acceptable if feature not in scope
  ✓ All SKIP results documented
  ✓ All FAIL results escalated with console logs

HOW TO RUN
================================================================================

Manual Testing:
  1. Open gap-suites-AE-AG.md in your QA tool (TestRail, Jira, etc.)
  2. For each TC:
     - Follow navigation steps
     - Execute test steps in order
     - Record result (PASS/FAIL/SKIP/GAP)

Automated Testing with Playwright:
