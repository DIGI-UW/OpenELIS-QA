The Inventory storage seed case

The Inventory module is the canonical example that motivated this dashboard. The list view renders, items CRUD works, the receive-lot modal opens. But POST /rest/inventory-storage-locations returns 500 (BUG-40), so storage locations can’t be created. The ?isActive=true filter is silently ignored (BUG-41), so “active items” shows deactivated ones. POST /rest/inventory/reports/generate 404s (BUG-45), so the six report types are vapor. A lab inventory manager would call this M1 in 30 seconds; the prior render-PASS criterion called it “working with workaround.”

About this dashboard

This is the v6.1 baseline maturity dashboard for OpenELIS Global, hand-encoded from the QA history in SKILL.md, the bug table (Section 8), the partial-feature audit seed list (Section 8.5), and the QA reports from March–May 2026 against testing.openelis-global.org and mgtest.openelis-global.org.

How to read it

Modules are grouped by functional area. Each module’s rating is the lowest M-level any of its sub-features hits. A module with a perfect form-render but a broken POST is M1, not M2. A module with a working CRUD but no cross-module link verification is M2, not M3. Click any feature card for the evidence.

How to update it

The FEATURES array in the script block at the bottom of this file is the data source. Add new modules, change ratings, append bug numbers there. For programmatic generation from QA reports, see skill-sharpening-workplan-2026-05-12.md Phase A2.

What it’s not

Not live data. Not a replacement for qa-dashboard.html (which tracks test pass-rate over time). Not exhaustive — the 87 features here cover the major surfaces, but there are configuration screens and minor admin pages omitted for legibility.
